## Мнения из сообщества

### @vibecoder1

> Перестал бороться с отсутствием памяти — начал использовать это как плюс. Каждая сессия свежая, без накопленных ошибок. Главное — хороший CLAUDE.md и понятные коммиты.

Смена перспективы: отсутствие памяти = отсутствие «плохих привычек».

### @vibecoder2

> Веду файл TASKS.md в корне проекта. В начале каждой сессии пишу: "Прочитай CLAUDE.md и TASKS.md, потом скажи что понял." Агент быстро входит в контекст.

Практический подход: явный файл с текущим состоянием задач.

### @developer1

> Самый мощный контекст — git история. Пишу подробные коммиты: почему изменение, а не что изменение. Агент читает `git log` и понимает эволюцию проекта.

Git как долгосрочная память проекта.

### @senior_dev1

> В конце большой сессии прошу агента: "Напиши handoff-комментарий: что сделано, что осталось, какие решения приняты и почему." Сохраняю в TASKS.md. Следующая сессия начинается с этого.

Структурированный handoff как протокол между сессиями.

### @vibecoder3

> Для долгих проектов — отдельный файл DECISIONS.md. Архитектурные решения, почему выбрали X вместо Y. Без этого агент (и я сам) через месяц не понимаем зачем всё так сделано.

ADR (Architecture Decision Records) для AI-проектов.

## Код

```markdown
# TASKS.md — текущее состояние проекта

## Статус: 2026-03-27

### Сделано
- [x] Настроена авторизация через Supabase
- [x] Базовый CRUD для пользователей
- [x] Деплой на Vercel

### В процессе
- [ ] Страница профиля пользователя (50%)
  - Форма готова, нужна валидация
  - Загрузка аватара не работает

### Решения
- Выбрали Supabase вместо Prisma: нужен realtime
- Tailwind v4 вместо v3: новый проект, берём актуальное

### Следующий шаг
Доделать валидацию формы профиля, потом аватар
```

```bash
# Начало новой сессии
claude "Прочитай CLAUDE.md и TASKS.md. Скажи что понял о проекте и текущем состоянии."

# Конец сессии
claude "Обнови TASKS.md: что сделали сегодня, что осталось, какие решения приняли."
```

## Итог

Контекст живёт в файлах, не в памяти агента:

1. **CLAUDE.md** — постоянный контекст проекта (стек, правила)
2. **TASKS.md** — текущее состояние задач (обновлять каждую сессию)
3. **Git-история** — эволюция кода и решений
4. **Комменты в коде** — «почему» для неочевидной логики
