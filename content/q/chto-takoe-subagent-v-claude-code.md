## Мнения из сообщества

### @subagent_user1

> Ключевое в субагентах — изоляция контекста. Субагент делает поиск через Exa, собирает данные, возвращает выжимку. Основной агент получает только результат, а не весь «мусор» промежуточных шагов. Контекст чистый.

Изоляция контекста — главное, не только экономия.

### @cost_optimizer

> Haiku в 10 раз дешевле Opus. Для рутинных задач (парсинг, форматирование, поиск) субагент на Haiku — очевидный выбор. Opus тратишь только на архитектурные решения.

Экономия 10x на рутинных задачах — реальная.

### @parallel_user

> Запустил трёх субагентов параллельно: один парсил данные, второй форматировал, третий искал через Exa. Пока основной агент планировал следующий шаг. Скорость x3.

Параллельность — следующий уровень после последовательных субагентов.

### @council_user

> Совет директоров через субагентов: CTO, CMO, Product Lead — каждый отдельная сессия со своим системным промптом. Параллельно анализируют проект, возвращают оценки. Синтезирую лучшие идеи.

Архитектура «совет директоров» — популярный паттерн.

### @when_not_use

> Субагент не нужен когда: задача простая и решится за один шаг, или задача требует полного контекста проекта (субагент его не видит). Оверинжиниринг тоже бывает.

Не каждая задача требует субагента.

## Итог

**Когда нужен субагент:**
- Рутинная работа: поиск, парсинг, форматирование
- Параллельные независимые задачи
- Нужна чистота основного контекста
- Экономия: рутина на Haiku, планирование на Opus

**Паттерны:**

```
Контент-фабрика:
Opus (планирует темы)
    ├── Haiku: собирает источники
    ├── Haiku: пишет черновики
    └── Opus: финальная редактура
```

**Субагент vs основной агент:**

| | Основной | Субагент |
|--|----------|----------|
| Модель | Opus (умный) | Haiku (быстрый) |
| Контекст | Полный проект | Только задача |
| Задачи | Планирование | Рутина |
| Стоимость | ~$15/млн токенов | ~$1.5/млн токенов |

**Когда НЕ нужен:** задача простая, или требует видеть весь проект целиком.
