## Мнения из сообщества

### @vibecoder1

> Месяц пользовался только Opus — думал, что для кода нужна максимальная модель. Переключился на Sonnet — разницы почти нет для обычных задач, а скорость выросла в 2-3 раза. Сейчас Opus только для архитектурных решений.

Практический вывод от реального использования.

### @vibecoder2

> Sonnet + хороший промпт > Opus + плохой промпт. Качество контекста важнее качества модели для большинства задач.

Принцип: промпт важнее выбора модели.

### @developer1

> Использую Opus когда нужно: 1) спроектировать архитектуру с нуля, 2) отладить сложную многофайловую проблему, 3) разобраться в чужом коде. Всё остальное — Sonnet.

Конкретные сценарии для Opus.

### @senior_dev1

> Haiku для простых задач типа "переименуй переменную", "напиши комментарий", "отформатируй". Дёшево и быстро. Sonnet для реального кода. Opus редко.

Трёхуровневая стратегия выбора модели.

### @vibecoder3

> Opus стал моим "старшим консультантом". Не для написания кода, а для ответа на вопрос "а правильно ли я вообще подхожу к этой задаче?" Раз в день максимум.

Метафора: Opus как эксперт для критических решений.

## Когда какую модель

| Задача | Модель | Почему |
|--------|--------|--------|
| Написать функцию | Sonnet | Достаточно мощный, быстро |
| Починить баг | Sonnet | Хорошо справляется |
| Рефакторинг файла | Sonnet | Нормальная задача |
| Спроектировать архитектуру | Opus | Нужно глубокое рассуждение |
| Разобрать сложный легаси-код | Opus | Много контекста и логики |
| Переименовать переменную | Haiku | Не нужна мощность |
| Написать README | Haiku | Простой текст |

## Итог

**Sonnet** — дефолт для вайбкодинга. Быстрый, умный, доступный по Pro-подписке.
**Opus** — когда задача требует максимального рассуждения: архитектура, сложная отладка, стратегические решения.
**Haiku** — для механических задач, когда нужна скорость и цена.

Переплачивать за Opus на каждый коммит не нужно.
