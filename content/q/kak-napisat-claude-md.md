## Мнения из сообщества

### @vibecoder1

> Потратил час на нормальный CLAUDE.md — сэкономил недели объяснений. Теперь агент сразу знает: TypeScript strict, Prisma, тесты через vitest. Не надо повторять каждый раз.

Главная идея: один хороший файл заменяет постоянное введение в контекст.

### @vibecoder2

> Самая важная секция — запреты. Я пишу "Не трогать public/, не менять schema.prisma без подтверждения, не использовать any в TypeScript." Это спасает от дорогих ошибок.

Практический акцент: ограничения важнее описаний.

### @vibecoder3

> Структура, которая работает: Stack (что используем), Commands (как запускать), Architecture (где что лежит), Rules (что нельзя). Всё, больше ничего не нужно.

Минималистичный фреймворк для любого проекта.

### @senior_dev1

> Добавляю раздел "Testing" с командами. Агент сам запускает тесты после изменений. Без этого раздела — не запускает.

Специфичная деталь: явно написанные команды = автоматическое их выполнение.

## Код

```markdown
# Мой проект

## Stack
- Next.js 15 + TypeScript strict
- Supabase (Postgres + Auth)
- Tailwind CSS
- Vitest для тестов

## Commands
| Task | Command |
|------|---------|
| Dev | `npm run dev` |
| Test | `npm run test` |
| Build | `npm run build` |
| DB migrate | `npx supabase db push` |

## Architecture
| Path | Purpose |
|------|---------|
| `app/` | Next.js App Router |
| `lib/` | Утилиты и хелперы |
| `components/` | React компоненты |
| `prisma/` | Схема БД |

## Rules
- ✅ Всегда запускать тесты после изменений
- ⚠️ Спрашивать перед изменением schema.prisma
- 🚫 Никогда не использовать `any` в TypeScript
- 🚫 Не редактировать .env напрямую
```

## Итог

Хороший CLAUDE.md содержит:
1. **Стек** — язык, фреймворки, версии
2. **Команды** — как запускать, тестировать, деплоить
3. **Архитектуру** — где что лежит и зачем
4. **Правила** — что делать всегда, что спрашивать, что никогда
5. **Контекст** — особенности проекта, которые не очевидны

Обновляй файл каждый раз, когда объясняешь агенту одно и то же второй раз.
