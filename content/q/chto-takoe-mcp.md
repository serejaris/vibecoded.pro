## Мнения из сообщества

### @dev1

> До MCP каждая интеграция требовала написания кастомного кода. Теперь подключаешь готовый сервер одной командой — и агент сразу умеет делать веб-поиск или управлять браузером.

Принципиальный сдвиг: не «написать интеграцию», а «подключить стандартный сервер».

### @dev2

> Первый MCP, который стоит подключить — EXA. Агент сам решает когда искать в интернете, не нужно ничего дополнительно настраивать в промптах.

```bash
claude mcp add --transport http exa https://mcp.exa.ai/mcp
```

### @dev3

> Важный совет: не подключайте больше двух MCP-серверов одновременно. Каждый сервер занимает место в контекстном окне. При трёх серверах теряете 10% контекста на их описания.

Правило двух серверов — практическое, не теоретическое.

### @dev4

> OpenAI и Google DeepMind тоже приняли стандарт. MCP стал отраслевым — не только для Claude, но и для других агентов.

Инвестиция в MCP — это не Anthropic lock-in, а универсальный навык.

### @dev5

> Популярный стек: EXA (поиск) + Playwright (браузер). EXA для исследований, Playwright для тестирования фронтенда и автоматизации.

Два сервера — максимум. Этот стек покрывает 80% задач.

## Итог

**Как работает MCP:**

```
Claude Code (MCP-клиент)
    │
    ├── EXA MCP ─────── веб-поиск
    ├── Playwright MCP ─ управление браузером
    └── Postgres MCP ─── запросы к БД
```

**Подключить за минуту:**

```bash
# Добавить EXA (рекомендуется первым)
claude mcp add --transport http exa https://mcp.exa.ai/mcp

# Проверить
claude mcp list
```

**Популярные серверы:**
- **EXA** — веб-поиск по смыслу, must-have
- **Playwright** — автоматизация браузера
- **Postgres** — запросы к БД
- **GitHub** — issues, PR, репозитории
- **Context7** — актуальная документация библиотек
