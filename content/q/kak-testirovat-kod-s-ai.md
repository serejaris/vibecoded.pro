## Мнения из сообщества

### @tdd_user

> TDD с AI перевернул процесс. Раньше: писал код → тест показывал баг → фиксил. Теперь: описываю что хочу → агент пишет тест → пишет реализацию под тест → всё зелёное. Меньше итераций.

TDD с AI — быстрее, чем TDD без AI.

### @repair_loop

> Repair loop в Claude Code: написал функцию → запустил pytest → 2 теста упали → агент читает ошибки → фиксит → запускает снова. Всё без моего участия. Я только проверяю финальный результат.

Repair loop — главная ценность агентного тестирования.

### @hook_tests

> Добавил хук: после каждого сохранения Python-файла запускать pytest для этого модуля. Теперь тесты запускаются автоматически, агент сразу видит результат и при необходимости фиксит.

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "write_to_file",
      "command": "pytest tests/ -x -q 2>&1 | tail -20"
    }]
  }
}
```

### @coverage_user

> Попросил агента написать тесты для существующего кода с покрытием >80%. Он сам разобрался в архитектуре, написал unit и integration тесты, настроил pytest. Покрытие с 0 до 85% за один запрос.

### @regression_user

> Главный юзкейс тестов в вайбкодинге — защита от регрессий. Агент добавляет фичу → тесты проверяют что старое не сломалось. Без тестов никогда не знаешь что сломал.

Тесты для вайбкодинга — страховка, не академическая практика.

## Итог

**Как попросить агента написать тесты:**

```
Напиши тесты для модуля auth.py:
- unit тесты для каждой функции
- integration тест для полного flow регистрации
- покрытие >80%
- pytest, без mock там где возможно
```

**Автоматический repair loop:**
1. Агент меняет код
2. Запускает тесты через хук или явно
3. Тест упал → агент читает ошибку
4. Фиксит → снова запускает
5. Всё зелёное → отчитывается

**Хук для автоматических тестов:**

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "write_to_file",
      "command": "pytest tests/ -x -q 2>&1 | tail -20"
    }]
  }
}
```

**Что агент умеет без инструкций:**
- Писать pytest тесты по существующему коду
- Запускать тесты и читать результаты
- Фиксить код на основе упавших тестов
- Настраивать покрытие и CI
