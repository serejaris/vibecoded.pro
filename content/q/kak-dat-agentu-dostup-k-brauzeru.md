## Мнения из сообщества

### @vibecoder1

> Playwright MCP — game changer для UI-разработки. Показываю агенту задачу, он сам открывает браузер, видит результат, исправляет. Больше не нужно описывать словами "кнопка слева, текст красный" — он просто видит.

Визуальный фидбек как новый уровень возможностей.

### @developer1

> Использую для E2E тестов. Прошу агента: "Открой localhost:3000, зайди как пользователь, добавь товар в корзину, оформи заказ. Скажи что видишь на каждом шаге." Агент проходит весь флоу и репортит проблемы.

Функциональное тестирование через браузерный агент.

### @vibecoder2

> Осторожно с авторизованными сайтами. Если агент кликает что-то в браузере с твоей сессией — он может случайно что-то отправить/удалить. Для проверки UI лучше localhost, не прод.

Предупреждение о безопасности: браузерный агент действует от твоего имени.

## Код

```json
// ~/.claude/settings.json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

```bash
# После настройки — в сессии Claude Code
claude "Открой http://localhost:3000, сделай скриншот главной страницы"

claude "Открой /dashboard, войди как test@example.com с паролем test123,
        проверь что таблица данных загружается правильно"

claude "Зайди на /signup, заполни форму невалидными данными,
        проверь что ошибки валидации показываются"
```

## Что умеет агент с браузером

| Возможность | Пример |
|------------|--------|
| Открыть страницу | Проверить что деплой прошёл |
| Сделать скриншот | Показать как выглядит UI |
| Кликнуть элемент | Пройти флоу регистрации |
| Заполнить форму | Протестировать валидацию |
| Прочитать текст | Проверить содержимое страницы |
| Проверить консоль | Найти JavaScript ошибки |

## Итог

Playwright MCP — один из самых полезных MCP-серверов для вайбкодинга. Устанавливается за минуту, сразу даёт агенту глаза. Особенно ценно для UI-задач и E2E тестирования без написания тестового кода вручную.
