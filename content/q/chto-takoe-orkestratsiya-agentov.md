## Мнения из сообщества

### @orchestration_user1

> Оркестрация — это переход от «один агент делает всё» к «умный агент планирует, быстрые агенты исполняют». Дешевле, быстрее, чище контекст.

Разделение ответственности между агентами — аналог микросервисов.

### @content_factory

> Контент-фабрика на оркестрации: Haiku собирает темы из интернета → Haiku делает черновики → Opus редактирует финальные версии. Один Opus для всего был бы в 10 раз дороже.

Оркестрация = оптимизация стоимости + качества одновременно.

### @council_user

> Совет директоров — мой любимый паттерн. Один промпт запускает трёх субагентов параллельно: CTO оценивает технический долг, CMO — позиционирование, Product Lead — приоритеты. За 5 минут получаешь три точки зрения.

Параллельный анализ с разных позиций — невозможен без оркестрации.

### @pipeline_user

> Pipeline для исследований: EXA ищет → Haiku суммирует → Opus делает выводы → публикует. Каждый шаг специализирован, результат передаётся дальше. Полный исследовательский цикл без участия человека.

### @scale_user

> Путь масштабирования: чат-бот → агент с инструментами → агент + субагенты → оркестрация → личная корпорация. Каждый шаг — новый уровень автономности.

## Итог

**Паттерны оркестрации:**

```
Orchestrator-Worker:
Opus (оркестратор)
    ├── Haiku: парсинг данных
    ├── Haiku: форматирование
    └── Haiku: поиск через Exa

Совет директоров (параллельно):
    ├── Claude как CTO: технический анализ
    ├── Claude как CMO: маркетинг
    └── Claude как Product: приоритеты

Pipeline (последовательно):
Поиск → Черновик → Редактура → Публикация
```

**Когда применять:**
- Рутинная работа, которую можно делегировать дешёвой модели
- Задачи с несколькими независимыми аспектами
- Конвейер с последовательными шагами
- Стратегический анализ с нескольких точек зрения

**Экономика:** Haiku в 10x дешевле Opus. Оркестрация = Opus только там, где нужен ум.
