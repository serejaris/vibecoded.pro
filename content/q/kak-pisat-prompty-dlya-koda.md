## Мнения из сообщества

### @prompt_master

> Главный сдвиг: перестать писать «сделай форму» и начать писать «создай форму регистрации: email, пароль, подтверждение пароля. Валидация на клиенте через Zod. React + shadcn/ui. Стиль — минималистичный».

Разница в количестве итераций: 1 vs 5.

### @context_provider

> Контекст стека важнее любой другой части промпта:
> ```
> Проект: Telegram-бот на python-telegram-bot 20.x
> БД: PostgreSQL на Railway
> Стиль: async/await, type hints обязательны
>
> Задача: добавь /stats с количеством сообщений за 7 дней
> ```
> С таким контекстом агент не придумывает архитектуру — он следует существующей.

### @error_prompter

> «Ошибка тоже промпт» — лучший совет для новичков. Не описываешь ошибку своими словами — просто копируешь весь стектрейс и вставляешь. Агент разберётся сам.

Полный стектрейс лучше краткого описания в 100% случаев.

### @few_shot_user

> Few-shot работает магически:
> ```
> Формат:
> Вход: "купить молоко" → Категория: Продукты
> Вход: "позвонить врачу" → Категория: Здоровье
>
> Теперь: "оплатить аренду"
> ```
> Пример показывает нужный формат лучше любого объяснения.

### @iterate_mindset

> Первый результат не должен быть идеальным — это неэффективно. Цикл: промпт → результат → «измени X, добавь Y, убери Z» → результат. Итерация дешевле переписывания промпта.

Итерация — не провал, а рабочий процесс.

## Итог

**Чеклист хорошего промпта:**

1. **Конкретная задача** — не «сделай форму», а «создай форму с полями X, Y, Z»
2. **Стек и версии** — Python 3.12, FastAPI, PostgreSQL
3. **Ограничения** — async/await везде, не трогать production.yml
4. **Ожидаемый формат** — если нужен конкретный вывод, покажи пример
5. **Контекст ошибки** — полный стектрейс, не краткое описание

**Примеры трансформации:**

| Плохой промпт | Хороший промпт |
|--------------|----------------|
| Сделай форму | Создай форму регистрации: email + пароль, валидация Zod, React + shadcn/ui |
| Есть ошибка | `[полный стектрейс]` + «ожидаю X, получаю Y» |
| Оптимизируй код | Оптимизируй запрос к PostgreSQL в функции `get_stats()`, сейчас 2+ секунды |

**Chain of Thought** — для сложных задач:
```
Проанализируй архитектуру. Шаг за шагом:
1. Какие компоненты есть
2. Как они связаны
3. Где потенциальные проблемы
```
