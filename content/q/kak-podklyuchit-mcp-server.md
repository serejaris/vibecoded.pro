## Мнения из сообщества

### @exa_user

> EXA — первый MCP для любого. Агент сам решает когда искать в интернете. Столкнулся с незнакомой библиотекой → сам нашёл документацию → применил. Не надо ничего в промпте — работает автоматически.

```bash
claude mcp add --transport http exa https://mcp.exa.ai/mcp
```

### @playwright_user

> Playwright MCP для фронтенда — обязательно. Агент открывает браузер, проверяет вёрстку, кликает по кнопкам, пишет тесты. Всё без твоего участия. Особенно ценно для responsive design.

### @context7_user

> Context7 даёт агенту актуальную документацию библиотек в реальном времени. Без него агент иногда использует устаревшие API (его знания ограничены датой обучения). С Context7 — всегда актуальный синтаксис.

### @two_max_rule

> Проверил правило «максимум два MCP»: подключил три, и первые сообщения явно хуже — агент тратит место в контексте на описания инструментов. Два — оптимум.

Правило двух — практическое ограничение, не произвол.

### @local_mcp

> Локальные MCP через npm тоже работают:
> ```bash
> claude mcp add exa -- npx -y exa-mcp-server
> ```
> Но hosted версия (через `--transport http`) проще и не требует API-ключа.

Hosted > local для большинства серверов.

## Итог

**Быстрый старт:**

```bash
# Добавить EXA (рекомендуется первым)
claude mcp add --transport http exa https://mcp.exa.ai/mcp

# Проверить список
claude mcp list

# Удалить если нужно
claude mcp remove exa
```

**Топ серверов для вайбкодинга:**

| Сервер | Что делает | Приоритет |
|--------|-----------|-----------|
| EXA | Веб-поиск по смыслу | Must-have |
| Playwright | Браузер, тестирование UI | Фронтенд |
| Postgres | Запросы к БД | Бэкенд |
| Context7 | Актуальная документация | Всегда актуально |
| GitHub | Issues, PR, репозитории | Работа с кодом |

**Правила:**
- Максимум 2 сервера одновременно
- EXA всегда первым
- Второй — по задаче (Playwright для UI, Postgres для БД)
