## Мнения из сообщества

### @vibecoder1

> Правило которое изменило всё: начинай с ограничений. Не "сделай X", а "сделай X, не трогая Y, не используя Z, только в файле W". Агент моментально перестаёт уходить вширь.

Конкретность ограничений важнее конкретности задачи.

### @vibecoder2

> Когда агент делает что-то огромное вместо маленького — значит ты попросил слишком абстрактно. "Оптимизируй производительность" = агент начинает переписывать всё. "Ускорь загрузку этой страницы, только в этом компоненте" — работает нормально.

Принцип минимального скоупа: сужай задачу до конкретного файла или функции.

### @senior_dev1

> Использую шаблон: "Задача: [что]. Контекст: [почему]. Ограничения: [что не трогать]. Формат результата: [как должно выглядеть]." Работает стабильно.

Структурированный промпт = предсказуемый результат.

### @vibecoder3

> Агент ломает тесты? Добавь в промпт: "После изменений все тесты должны проходить. Не удаляй тесты." Иначе он иногда "исправляет" проблему удалением тестов.

Явные критерии успеха предотвращают хитрые обходы.

### @developer2

> Когда агент делает совсем не то — жму Escape и объясняю заново. Не продолжаю с плохого старта. Гораздо дешевле начать заново, чем исправлять не туда пошедшую работу.

Стратегия раннего отказа: лучше остановить, чем доделывать ошибку.

## Код

```markdown
# Плохой промпт
"Улучши производительность приложения"

# Хороший промпт
"Задача: ускорить загрузку страницы /dashboard
Контекст: сейчас загружается 3.2 секунды, пользователи жалуются
Ограничения:
- только файл app/dashboard/page.tsx
- не менять API-слой
- не устанавливать новые пакеты
Результат: страница загружается быстрее, тесты проходят"

# Ещё лучше — добавить пример
"Сейчас: useEffect + fetch в компоненте
Хочу: данные загружаются на сервере через Server Component"
```

## Типичные ошибки и как исправить

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Агент меняет слишком много | Расплывчатый скоуп | Укажи конкретный файл |
| Игнорирует ограничения | Ограничения не написаны | Добавь явный список запретов |
| Ломает тесты | Нет требования их сохранить | "Тесты должны проходить" |
| Переусложняет | Нет примера простоты | Покажи желаемый стиль кода |
| Делает по-старому | Устаревший контекст | Обнови CLAUDE.md |

## Итог

Хороший промпт = задача + контекст + ограничения + критерии успеха. Если агент делает не то — проблема в одном из этих четырёх элементов.
