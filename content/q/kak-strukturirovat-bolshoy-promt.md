## Мнения из сообщества

### @vibecoder1

> Главная ошибка: давать агенту задачу на 2 часа работы в одном промпте. Он либо теряет контекст, либо делает первые части хорошо, а к концу начинает импровизировать. Разбивай на части по 15-30 минут работы.

Принцип размера задачи.

### @vibecoder2

> Двухшаговый подход: сначала "составь план выполнения задачи", потом "выполни шаг 1". Между шагами проверяю результат. Если что-то не то — исправляю направление до следующего шага, не после всей работы.

Plan-then-execute как базовый паттерн.

### @senior_dev1

> Для архитектурных задач прошу агента задавать мне вопросы перед началом: "Прежде чем начать, задай мне уточняющие вопросы о требованиях." Агент спрашивает, я отвечаю — и потом он делает именно то что нужно.

Сократический подход: агент интервьюирует пользователя.

### @developer1

> Документ с требованиями работает лучше длинного промпта. Пишу requirements.md, даю агенту прочитать, прошу подтвердить что понял. Потом "сделай пункт 3 из requirements.md."

Внешний документ требований как источник истины.

## Код

```markdown
# Шаблон сложного промпта

## Контекст
[Что это за проект, какой стек, где находятся нужные файлы]

## Задача
[Что конкретно нужно сделать — одно конкретное изменение]

## Требования
1. [Конкретное требование]
2. [Конкретное требование]
3. [Что НЕ нужно трогать]

## Критерий готовности
[Как поймём что задача выполнена — тест, скриншот, поведение]
```

```bash
# Шаг 1: Получить план
claude "Мне нужно добавить систему уведомлений в приложение.
Прочитай CLAUDE.md, потом составь план реализации по шагам.
Не начинай код — только план."

# Шаг 2: Выполнить первый шаг плана
claude "Хорошо, выполни шаг 1 из плана: создай модель Notification в базе данных."

# Шаг 3: Проверить и продолжить
claude "Миграция прошла успешно. Выполни шаг 2: API endpoint для создания уведомления."
```

## Итог

Правила структурирования больших задач:
1. **Сначала план** — попроси агента составить план, проверь его
2. **Шаг за шагом** — выполняй по одному шагу с проверкой
3. **Конкретный скоуп** — каждый промпт = одно изменение
4. **Критерий готовности** — как поймёшь что шаг выполнен

Большая задача — это просто много маленьких задач последовательно.
