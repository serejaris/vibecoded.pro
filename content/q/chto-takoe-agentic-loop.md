## Мнения из сообщества

### @agent_user1

> Первый раз понял разницу, когда попросил Claude Code добавить авторизацию в проект. Он не спросил «как именно?» — просто начал читать файлы, планировать, писать, тестировать. Всё сам, по кругу.

Агент не ждёт разрешения на каждый шаг — он действует до результата.

### @agent_user2

> Агентский цикл можно визуализировать как паттерн ReAct: Reason + Act. Думает о следующем шаге → делает → смотрит на результат → снова думает. ChatGPT в обычном чате этого не делает.

ReAct — базовый паттерн за всеми современными агентами.

### @context_problem

> Главное ограничение цикла — контекстное окно. Качественно агент работает 1-2 часа. Потом начинает «плыть»: повторяет действия, забывает что решил в начале. Признак — пора начинать новую сессию.

Длинные задачи надо разбивать на сессии, фиксируя прогресс.

### @loop_problem

> Столкнулся с зацикливанием: агент пытался починить баг, создавал новый, чинил его, создавал следующий. Решение — новая сессия с чётким описанием проблемы. Или включить human-in-the-loop.

Зависание в цикле — известная проблема. Распознать и перезапустить.

### @codex_user

> Codex — пример полного автономного цикла: берёт GitHub Issue, создаёт ветку, пишет код, запускает тесты, открывает Pull Request. Ты только проверяешь PR в конце.

Уровень автономности зависит от инструмента и настроек.

## Итог

**Как выглядит агентский цикл:**

```
Задача: "добавь авторизацию"
    ↓
Читает проект → планирует → пишет код
    ↓
Запускает тесты → тест упал
    ↓
Читает ошибку → фиксит баг
    ↓
Тесты прошли → отчитывается
```

**Когда агент начинает «плыть» (пора в новую сессию):**
- Повторяет одни и те же действия
- Забывает решения начала сессии
- Качество кода снижается

**Как продлить сессию:**
- Разбивать большие задачи на части
- Фиксировать прогресс в GitHub Issues или CLAUDE.md
- Использовать хуки для автоматической записи контекста
